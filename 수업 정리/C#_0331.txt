3/31 수업 정리

[ 키워드 ]
- 일반 키워드(Keyword) : 예약어(키워드). 변수명 등으로 사용 불가능
  * C#에서는 string <- s가 소문자! (예약어)
- 컨텍스트 키워드(Context Keyword, 문맥키워드) : 어떤 부분에서는 의미를 가진다
- 데이터 추출에 있어서 C#이 매우 강력하다! (깔끔하고 간결)

(기호 용어 정리)
_ : 언더 스코어, 언더 바
- : 하이픈, 데시
| : 파이프, 바


[ 식별자 ]
- 규칙은 자바와 거의 비슷
- 속성, 메서드, 네임 스페이스(패키지)들의 이름도 대문자로 시작  <- 자바와 다르다
- 지역/전역 변수 이름은 소문자로 시작  <- *전역 변수는 없는 듯이 해야한다 (유지보수가 매우 힘듬)


[ 주석 ]
- 주석에서는 코드의 의도, 코드만 보고는 모르는 것들을 적는 것이 가장 이상적이다 (코드 설명 길게 적기 보다는)
- annotation(@)도 번역하면 주석이지만, 컴파일러를 위한 주석이다. (일반 주석과 용도가 다름)
- 일반 주석 기호를 사용한 주석은 컴파일 시 전부 사라진다.


[ 출력문 ]
Console.WriteLine(); -> 출력 후 개행이 된다
Console.write();      -> 출력 후 개행이 되지 않는다
